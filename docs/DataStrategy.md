## 0. 문제 정의

학원 데이터는 에듀픽의 핵심 자산이자 가장 큰 진입 장벽이다.
좋은 데이터 없이는 탐색 기능이 작동하지 않고, 탐색이 없으면 강사 온보딩도 안 된다.

### 학원 데이터의 특성 (왜 어려운가)
- **파편화**: 네이버 블로그, 카카오맵, 인스타, 입소문에 분산
- **비표준화**: 학원마다 시간표, 가격, 셔틀 공지 형식이 다름
- **휘발성**: 수강료·시간표·강사가 자주 바뀜
- **신뢰 문제**: 광고성 정보와 실제 정보 구분 어려움
- **Long tail**: 소규모 개인 강사는 온라인 존재감이 거의 없음

### 수집해야 할 데이터 항목

```
기본 정보       학원명, 주소, 전화, 카테고리, 설립일
수업 정보       과목, 연령대, 시간표, 정원, 수강료
강사 정보       이름, 경력, 자격증, 수업 스타일
시설/편의       주차, 셔틀 노선, 정류장 위치/시간
미디어          사진, 동영상, 커리큘럼 PDF
신뢰 지표       재원생 수, 리뷰, 영업 기간
정책            환불 규정, 결석 처리, 보강 정책
바우처          평생교육바우처, 문화누리카드 수락 여부
```

### 2.1 공공데이터 (Official Government Data)

#### 교육청 학원 등록 데이터 (공공데이터포털)

```
API: data.go.kr → "학원교습소정보"
엔드포인트: https://open.neis.go.kr/hub/acaInsTiInfo
제공 항목:
  - 학원명, 대표자, 주소, 전화번호
  - 교습 과목, 학원 구분 (일반/예능/체육)
  - 설립일, 정원, 수용 인원
  - 교습비 (신고 기준 — 실제와 다를 수 있음)
```

| 장점 | 단점 |
|---|---|
| 전국 모든 학원 등록 (법적 의무) | 실제 운영 시간표·수강료 미포함 |
| 무료 공공 API | 폐업 학원도 포함 (최신성 낮음) |
| 주소·카테고리 정확도 높음 | 개인 강사(미등록) 누락 |
| 법인 정보로 신뢰 가능 | 셔틀·강사 상세 정보 없음 |

**활용 전략**: 기초 DB 구축용. 주소·카테고리 뼈대로 사용하고, 나머지 상세 정보는 다른 소스로 보완.

**수집 방법**
```python
# NEIS 학원정보 API 예시
import requests

url = "https://open.neis.go.kr/hub/acaInsTiInfo"
params = {
    "KEY": "{API_KEY}",
    "Type": "json",
    "pIndex": 1,
    "pSize": 1000,
    "ATPT_OFCDC_SC_CODE": "B10",  # 서울특별시교육청
    "ADMST_ZONE_NM": "강남구"
}
response = requests.get(url, params=params)
```

**예상 커버리지**: 전국 학원의 약 80% (등록 학원 기준)

### 2.2 지도 플랫폼 데이터

#### 네이버 플레이스

대부분의 학원이 네이버 플레이스에 등록되어 있으며, 운영시간·사진·리뷰가 포함된다.

**공식 API 현황**
```
네이버 검색 API (search.naver.com):
  - 로컬 검색 지원 (학원명, 주소)
  - 응답: 학원명, 주소, 전화, 카테고리, URL
  - 제한: 일 25,000 콜, 상세 정보(시간표·가격) 미제공

네이버 플레이스 API (클로바):
  - 공개 API 없음 — 기업 제휴 필요
```

**크롤링 가능 범위 (법적 검토 필수)**
```
✅ 가능: 학원 자체 홈페이지 (공개 정보)
✅ 가능: 네이버 블로그 공개 포스팅 (학원 운영 블로그)
⚠️ 주의: 네이버 지도 크롤링 → robots.txt 위반 가능성
❌ 불가: 네이버 플레이스 리뷰 크롤링 (저작권)
```

**권장 접근법**: 네이버 검색 API로 학원명·주소 수집 → 에듀픽에 직접 등록 유도

---

#### 카카오 로컬 API

```
엔드포인트: https://dapi.kakao.com/v2/local/search/keyword.json
활용 가능:
  - 키워드 검색 (예: "강남구 수학 학원")
  - 위치 기반 반경 검색
  - 반환: place_name, address, phone, category, place_url, x/y 좌표

제한:
  - 일 30만 콜 (무료)
  - 상세 정보(시간표·수강료) 없음
  - 리뷰 데이터 없음
```

**활용 전략**: 지도 핀 표시용 좌표 + 기본 정보 수집. 카카오 API는 공식 사용 가능하므로 적극 활용.

```python
# 카카오 로컬 API 예시
headers = {"Authorization": f"KakaoAK {API_KEY}"}
params = {
    "query": "발레 학원",
    "x": "127.0495556",  # 강남구 중심
    "y": "37.5140229",
    "radius": 2000,       # 2km 반경
    "category_group_code": "AC5"  # 학원 카테고리
}
response = requests.get(
    "https://dapi.kakao.com/v2/local/search/keyword.json",
    headers=headers, params=params
)
```

### 2.3 SNS / 블로그 크롤링

#### 네이버 블로그 (학원 운영 블로그)

대부분의 학원은 네이버 블로그를 마케팅 채널로 운영한다.

**수집 가능 정보**
```
- 수업 사진 (시설 품질 파악)
- 커리큘럼 소개 (글에서 구조화 추출)
- 수강료 정보 (일부 공개)
- 강사 소개
- 학원 분위기·스타일
```

**수집 방법**
```python
# 네이버 블로그 검색 API 활용 (공식)
url = "https://openapi.naver.com/v1/search/blog.json"
headers = {
    "X-Naver-Client-Id": CLIENT_ID,
    "X-Naver-Client-Secret": CLIENT_SECRET
}
params = {"query": "역삼동 발레 학원", "display": 100}
# 반환: title, link, description, bloggername, postdate
```

**법적 고려사항**
```
✅ 네이버 검색 API 사용: 허용
✅ 블로그 공개 포스트 URL 수집: 허용
⚠️ 블로그 본문 전체 텍스트 크롤링: 저작권법 검토 필요
   → 요약 발췌(snippet) 수준은 대체로 허용 범위
   → 전문 저장은 저작권자 동의 필요
❌ 비공개 블로그, 회원 전용 카페: 불가
```

**실용적 접근**: 블로그 포스트에서 학원 연락처·주소·카테고리 추출 → 에듀픽 DB 기초 구축에만 활용. 콘텐츠 자체는 저장하지 않고 링크만 연결.

### 2.4 커뮤니티 데이터 (맘카페, 지역 밴드)

**현황**: 가장 신뢰도 높은 정보가 집중되어 있으나 접근이 가장 어려움.

| 플랫폼 | 특징 | 수집 가능 여부 |
|---|---|---|
| 네이버 카페 (맘카페) | 폐쇄형, 가입 승인 필요 | ❌ API 없음, 크롤링 ToS 위반 |
| 카카오 단체방 | 완전 폐쇄 | ❌ 불가 |
| 밴드(Band) | 지역 학부모 그룹 | ❌ API 제한적 |

**대안 전략**: 커뮤니티 정보를 크롤링하는 대신, **에듀픽 내부에서 동등한 가치의 정보를 생산**하는 것이 목표.
→ 영수증 인증 리뷰 시스템 = 맘카페의 신뢰도를 구조화된 형태로 재현

### 2.5 직접 온보딩 (가장 중요한 소스)

다른 어떤 방법보다 **강사/원장이 직접 입력한 데이터**가 가장 정확하고 풍부하다. 에듀픽의 핵심 전략.

**온보딩 채널**

| 채널 | 방법 | 예상 효율 |
|---|---|---|
| 인바운드 (자발적 등록) | 에듀픽 앱에서 직접 프로필 생성 | 낮음 (초기) |
| 오프라인 영업 | BD팀 강남 3구 직접 방문 | 높음 (초기 집중) |
| 학원 협회 제휴 | 협회 회원에게 일괄 가입 혜택 제공 | 중간 |
| 기존 수강생 초대 | 학부모가 "우리 학원 에듀픽에 없어요" 신고 | 중간 |
| 카카오 알림톡 캠페인 | 교육청 DB 학원명으로 SMS/알림톡 발송 | 중간 |

**강사 온보딩 인센티브**
```
첫 3개월 수수료 0%
"에듀픽 인증 강사" 뱃지 (탐색 결과 상위 노출)
첫 달 카카오 알림톡 발송 비용 무료 지원
```

### 3.2 중복 학원 병합 로직

교육청 DB + 카카오맵 + 강사 직접 등록이 같은 학원을 가리킬 때 병합해야 한다.

```python
# 학원 동일성 판단 기준 (퍼지 매칭)
def is_same_academy(a, b) -> bool:
    # 1. 전화번호 일치 → 높은 확신
    if a.phone == b.phone:
        return True

    # 2. 주소 + 이름 유사도
    address_sim = jaro_winkler(normalize_address(a.address),
                               normalize_address(b.address))
    name_sim = jaro_winkler(a.name, b.name)

    # 3. 좌표 거리 (50m 이내 + 이름 유사)
    if haversine(a.coords, b.coords) < 50 and name_sim > 0.85:
        return True

    return address_sim > 0.9 and name_sim > 0.8
```

### 3.3 데이터 완성도 스코어링

```python
def calculate_completeness_score(academy) -> int:
    score = 0

    # 기본 정보 (40점)
    if academy.name: score += 10
    if academy.address and academy.coordinates: score += 10
    if academy.phone: score += 10
    if academy.category: score += 10

    # 수업 정보 (30점)
    if academy.schedules: score += 15      # 시간표
    if academy.pricing: score += 10        # 수강료
    if academy.age_groups: score += 5      # 대상 연령

    # 풍부한 정보 (30점)
    if academy.photos and len(academy.photos) >= 3: score += 10
    if academy.shuttle_routes: score += 10
    if academy.reviews and len(academy.reviews) >= 3: score += 10

    return score  # 0~100

# 검색 노출 우선순위에 반영
search_score = (0.4 * completeness_score
              + 0.4 * review_score
              + 0.2 * distance_score)
```

## 4. 법적 검토 사항

### 4.1 크롤링 합법성 판단 기준

```
판단 순서:
1. robots.txt 확인 → Disallow된 경로는 수집 금지
2. ToS(이용약관) 확인 → 크롤링/자동 수집 금지 조항 여부
3. 수집 목적 → 학원 정보 서비스는 공익 목적으로 일정 부분 허용
4. 저작물 여부 → 단순 사실 정보(주소, 전화)는 저작권 비대상
                   사진, 리뷰 텍스트는 저작물 → 저장 시 동의 필요
```

| 데이터 유형 | 저작권 해당 | 수집 가능 여부 |
|---|---|---|
| 학원명, 주소, 전화번호 | ❌ (사실 정보) | ✅ 가능 |
| 운영 시간, 카테고리 | ❌ | ✅ 가능 |
| 수강료 | ❌ (공개 정보) | ✅ 가능 |
| 학원 자체 작성 소개 글 | ✅ | ⚠️ 요약만 가능, 전문 저장 시 동의 필요 |
| 사진 (학원 업로드) | ✅ | ❌ 동의 없이 저장 불가 |
| 카페/블로그 리뷰 | ✅ | ❌ 저작권자 동의 필요 |

### 4.2 개인정보보호법 (PIPA) 고려

```
수집 시 고려사항:
- 원장/강사 이름, 전화번호는 개인정보
- 공개된 비즈니스 정보(사업자 정보)는 상대적으로 완화
- 수집 사실 고지 및 삭제 요청 처리 창구 필요

권장:
→ 교육청 공공데이터는 이미 공개 정보 → 문제 없음
→ 강사 개인 연락처는 본인 동의 후에만 수집/표시
→ "우리 학원 정보 삭제 요청" 기능 구현 필수
```

## 5. 데이터 신선도 유지 전략

학원 정보는 수시로 변한다 (수강료·시간표·강사 교체).

### 5.1 자동 업데이트 탐지

```
방법 1: 주기적 재수집
  - 교육청 API: 월 1회 전체 동기화
  - 카카오 플레이스: 분기 1회 변경 감지 (영업 중단 여부)

방법 2: 사용자 신고 기반
  - "이 정보가 틀렸어요" 버튼 → 학원에 수정 요청 알림

방법 3: 강사 참여 유도
  - "정보 업데이트 하면 탐색 결과 상위 노출" 인센티브
  - 월 1회 "정보가 최신인지 확인해주세요" 알림

방법 4: 전화번호 생존 확인
  - 분기 1회 자동 전화 발신 테스트 (ARS)
  - 결번 → 폐업 의심 플래그 + 수동 확인
```

### 5.2 데이터 신뢰도 표시

```
탐색 결과 카드에 마지막 업데이트 날짜 표시
  "정보 업데이트: 2주 전" → 신뢰
  "정보 업데이트: 6개월 전" → 주의 아이콘

학원 직접 관리 시: "학원 인증 정보" 뱃지 표시
```

## 6. 단계별 실행 계획

### Phase 0 — 종자 데이터 구축 (출시 전, ~M0)

```
목표: 강남 3구 학원 2,000개 기초 DB 구축

Week 1~2:
  □ 교육청 NEIS API 연동 → 강남/서초/송파 학원 전수 수집
  □ 카카오 로컬 API → 좌표 및 운영 여부 보완
  □ 중복 병합 + 폐업 학원 제거

Week 3~4:
  □ 네이버 검색 API → 블로그 URL 수집 (학원별 1~3개)
  □ 카테고리 표준화 (교육청 과목 코드 → 에듀픽 카테고리 매핑)
  □ 완성도 스코어 산출

산출물: 강남 3구 학원 약 1,500~2,000개, 완성도 ⭐ 수준
```

### Phase 1 — 강사 직접 온보딩 (M1~M3)

```
목표: 종자 DB 기반 강사 직접 등록 유도 → ⭐⭐⭐ 데이터로 전환

방법:
  □ "우리 학원이 에듀픽에 있어요 — 정보를 완성해 주세요" 알림톡 발송
     (교육청 DB의 학원 전화번호 활용)
  □ BD팀 강남 3구 오프라인 방문 (우선: 발레·피아노·미술·영어)
  □ 강사 온보딩 완료 시 → 시간표·수강료·사진·셔틀 정보 입력
  □ 학원 협회 제휴 (한국학원총연합회, 발레협회 등) 협약 체결

목표: 강사 직접 등록 50명, 완성도 ⭐⭐⭐ 전환
```

### Phase 2 — 확장 및 자동화 (M4~M6)

```
목표: 수도권 전체로 확대 + 데이터 자동화

  □ 강사 인스타그램 연동 (수업 사진 자동 동기화)
  □ 학부모 리뷰로 미등록 학원 자동 발견
     ("리뷰 쓰려는 학원이 없어요" → 신규 등록 플로우)
  □ 바우처 가맹점 DB 연동 (평생교육바우처, 문화누리카드)
  □ 폐업 학원 자동 감지 파이프라인 가동
  □ 수도권 전체 학원 기초 DB 구축 (경기·인천 확장)
```

### Phase 3 — 전국 확장 (M7~)

```
  □ 5대 광역시 교육청 API 순차 연동
  □ "강사 커뮤니티" 기능으로 강사들의 자발적 정보 업데이트 유도
  □ 데이터 품질 리포트 자동 생성 (미완성 학원 대상 알림)
```

## 7. 경쟁 우위로서의 데이터 전략 요약

```
단기 (M0~M3): 공공 API로 뼈대 → 영업으로 살 붙이기
  교육청 DB로 커버리지 확보
  오프라인 영업으로 핵심 지역 ⭐⭐⭐ 달성

중기 (M4~M6): 사용자가 데이터를 만드는 구조
  학부모 리뷰 → 신규 학원 발견
  강사 인스타 연동 → 콘텐츠 자동 수집
  "정보 완성할수록 더 많이 노출" 인센티브

장기 (M7~): 데이터 해자 형성
  누적 리뷰 수 → 경쟁자가 단기간 복제 불가
  셔틀 노선 데이터 → 에듀픽만의 독점 자산
  실시간 시간표 → 강사가 직접 관리하는 살아있는 DB
```

> **핵심 원칙**: 크롤링으로 수집한 데이터는 '문 여는 용도'고, 강사가 직접 입력하고 학부모가 검증한 데이터가 에듀픽의 진짜 경쟁력이다. 데이터 수집보다 데이터 생산 구조를 만드는 것이 목표.
