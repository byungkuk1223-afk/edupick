## Phase 1: MVP (2026년 4~6월)

**목표**: 강사 50명 온보딩, 학부모 MAU 500

### M1 (2026년 4월) — 코어 인프라
- [ ] 카카오 소셜 로그인
- [ ] 역할 선택 (학부모 / 강사)
- [ ] 강사 프로필 등록 (학원명, 주소, 카테고리, 수업 시간표)
- [ ] 네이버 지도 기반 학원 탐색 (위치 기반, 카테고리 필터)
- [ ] 학원 상세 페이지
- [ ] 자녀 프로필 등록

### M2 (2026년 5월) — 결제 & 커뮤니케이션
- [ ] 인앱 결제 (포트원 연동: 카드, 카카오페이)
- [ ] 자동 인보이스 발급 (PDF)
- [ ] 강사 → 수강생 공지 발송 (앱 푸시)
- [ ] 카카오 알림톡 연동 (공지, 결제 안내)
- [ ] 미납 자동 리마인더

### M3 (2026년 6월) — 신뢰 & 리텐션
- [ ] 인증 리뷰 시스템 (앱 내 결제 자동 인증)
- [ ] 기본 알림 센터 (공지, 결제, 수업 확인)
- [ ] 강사 대시보드 (수강생 관리, 결제 현황)
- [ ] 학부모 대시보드 (오늘 일정, 결제 내역)
- [ ] **베타 출시** (강남 3구 집중)

**Phase 1 성공 기준**
- 강사 등록 50명 이상
- 학부모 MAU 500 이상
- D7 리텐션 60% 이상
- NPS 30 이상

## 기능 우선순위 매트릭스

```
높은 임팩트
    │
    │  [카카오알림톡]  [인증리뷰]
    │  [지도탐색]      [인앱결제]
    │  [시간표빌더]    [셔틀알림]
    │
    │  [펀딩]          [아이리포트]
    │  [가족공유]      [커뮤니티]
    │
낮은 임팩트
    ├────────────────────────────
    낮은 노력              높은 노력
```

**P0 (즉시 시작)**: 지도 탐색, 카카오 로그인, 기본 결제, 공지 발송, 카카오 알림톡
**P1 (Phase 1 완료 전)**: 인증 리뷰, 인보이스, 강사 대시보드
**P2 (Phase 2)**: 시간표 빌더, 셔틀 알림, 펀딩, 아이 리포트
**P3 (Phase 3)**: 커뮤니티, 광고 상품, AI 추천

## 의존성 다이어그램

```
카카오 로그인
    → 역할 선택
        → 강사 등록 → 수업 등록 → 인앱 결제
                                 → 공지 발송 → 카카오 알림톡
        → 자녀 등록 → 학원 탐색 → 수강 신청 → 결제
                               → 셔틀 확인 → 셔틀 알림
                    → 시간표 빌더 (고정일정 입력 필요)

인증 리뷰 → 결제 이력 필요
수업 개설 펀딩 → 강사 등록 + 결제 시스템
아이 리포트 → 강사 + 자녀 + 수강 등록 필요
커뮤니티 → 인증 리뷰 시스템 필요 (신뢰 기반)
```
